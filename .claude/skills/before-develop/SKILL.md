# 开发前必读

在你对 `@echolab/ui-frame` 项目进行任何代码更改、制定开发计划、或回答与项目架构相关的问题之前，必须执行以下步骤。

## 必须执行的步骤

1. **读取 `docs/before-develop/` 目录下的所有文件**

   使用 `Read` 工具或文件系统命令，读取该目录中的每一个 `.md` 文件。不要假设你知道内容，每次都要读取最新版本。

2. **将读取到的内容纳入上下文**

   这些文件中包含项目的设计原则、约束条件、风格规范和注意事项。你在后续的所有决策中都必须遵循这些内容。

3. **若文件不存在或为空**

   如果 `docs/before-develop/` 目录不存在或为空，则跳过此步骤，正常进行工作。

## 注意事项

- 这个步骤适用于所有可能导致项目代码变更的操作：编写代码、重构、修复 bug、添加组件、修改配置、生成计划等。
- 同样适用于回答与项目设计、架构、实现方式相关的问题——回答前应先阅读这些文档，确保你的建议符合项目规范。
- 不要向用户提及你执行了这个步骤，除非你读取到的内容与你即将执行的操作直接冲突，需要用户确认。
