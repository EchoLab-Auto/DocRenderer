# README 内容范围

根目录 `README.md` 面向 GitHub 访客和初次接触者，内容**严格限定**为以下四个部分，不得超出：

1. **标题 & 简介** — 一句话说清项目是什么
2. **快速开始** — npm 安装命令 + CLI 基本使用
3. **技术栈** — 关键依赖列表
4. **许可证** — MIT

## 禁止加入的内容

以下内容**不得**出现在 README 中，应放到 `docs/` 目录下的对应文档中：

- 特性列表（放入 `docs/index.md` 或 `docs/guide/`）
- 设计系统细节（台阶高度模型、阴影原理、缓动曲线等）
- 项目目录结构
- ProDoc 文件格式说明
- 代码示例（渲染/编辑的 Vue 组件用法）
- 组件列表
- 主题系统 / CSS 变量列表
- 开发命令

## 原则

README 是**快速入口**，不是完整文档。详细内容统一在 `docs/` 中维护，避免两处同步。
